# 11 — Literature Module

Classical Chinese poems & prose (诗书) with search, filter, read-aloud (粵/國) and favouriting.
Integrated from the standalone `tiggerkk/chinese-literature` app — **rewritten** to WellWorth
conventions (dark design system, shared components, data layer); only the **read-aloud algorithm** and
the **corpus data** were carried over.

## Architecture — static corpus + Supabase favourites

- The **corpus** (poems, writers, types) is **immutable, shared, non-private reference data**, so it
  ships as a **versioned STATIC ASSET** under `public/literature/**` (not a Postgres table): fully
  offline, zero DB storage/egress, updates by redeploying files. This is the one corpus where RLS and
  cross-device sync add nothing.
- Only the **user-owned** state is in Supabase: `poem_favorite` (a join row) + two `profile` read-aloud
  columns — so favourites + settings sync across devices.
- **Search/filter is client-side** over the in-memory index (the genuine departure from other modules,
  correct because the data is static + bundled), using the always-resident `foldZh`
  (Traditional⇄Simplified-agnostic — `src/lib/zh-fold.ts`).
- **Dark design system only** — the source app's light cream/brown theme was not ported.

### Static asset layout (`public/literature/`, generated by `scripts/build-literature-data.mjs`)

- `meta.json` — `{ version, types[{id,name,kind,sortOrder}], writers[{id,name,dynasty}], dynasties[] }`.
  `kind ∈ theme|season|anthology|style|other`; the first four are shown as filters. **Precached.**
  The curated filter vocabulary (in `scripts/build-literature-data.mjs` `TYPE_GROUPS`) matches this
  corpus's tag spelling — 主題: 愛國/邊塞/懷古/羈旅/貶謫/哲理/景物/友情/愛情/孤獨/抒情/思念 · 時令:
  春天/夏天/秋天/冬天/節日 · 選集: 唐詩三百首/宋詞三百首/古詩三百首/古詩十九首/詩經/楚辭/古文觀止 +
  小學/初中/高中 各 古詩/文言文 · 風格: 婉約/豪放/樂府. Everything else (≈500 fine-grained gushiwen
  tags) stays `other`.
- **Alias layer (`TYPE_ALIAS_GROUPS`):** because most poems are tagged by fine-grained style/theme tags
  rather than the curated set, ~85 high-frequency `other` tags are **aliased** onto a curated parent
  (e.g. 寫景/山水/月亮 → 景物 · 中秋節/春節/… → 節日 · 抒懷/感慨 → 抒情 · 懷人/回憶 → 思念 · 民歌 → 樂府),
  so a poem tagged only with an alias still surfaces under its filter. The pills are unchanged — aliases
  only feed them. **The full curated lists + alias map are the source of truth in
  `scripts/build-literature-data.mjs` (`TYPE_GROUPS` + `TYPE_ALIAS_GROUPS`)** — edit there + rebuild to
  add/move a filter; this doc isn't an exhaustive copy. The unmapped long tail (misc tags like
  女子/生活/故事/讚美/飲酒) stays `other`; those poems remain reachable via the unfiltered list, search,
  and dynasty.
- **Tag coverage is source-limited.** Only **~20%** of poems (≈2,037 / 10,000) carry **any** type tag in
  the source `poems.db`; the other ~80% were never tagged by theme/form, so no mapping can reach them via
  a type filter (they're still found by search / dynasty / unfiltered browse). After aliasing, **≈95% of
  the _taggable_ poems** carry a filter tag — effectively the ceiling. The build prints `coverage:
N/10000` so this stays visible; chasing higher overall coverage means mis-filing, not better data.
- `index.json` — poem search/browse index: `[{ id, title, writerId, writer, dynasty, typeIds, excerpt }]`
  (excerpt = first 40 chars of HTML-stripped content, so a list card renders with no body fetch).
  **Precached** → browse/search/filter/favourites all work offline.
- `poem/<id>.json` — full body `{ …, content, translation, remark, shangxi, typeIds }`.
  **Runtime-cached** on open (CacheFirst, cache `literature-bodies-v1`).
- `writer/<id>.json` — `{ …, simpleIntro, detailIntro, headImageUrl, poemIds }`. **Runtime-cached.**

`poems.db` is the build input (gitignored at `scripts/literature/poems.db`); the generated
`public/literature/**` is committed (the deployable source of truth — no DB / `better-sqlite3` at
deploy/CI time). PWA wiring: precache via `includeAssets`; runtime-cache + `globIgnores` in
`vite.config.ts`. Favouriting a poem also `cache.add()`s its body so saved poems read offline (lazy
default otherwise caches only what's been opened). See `docs/OWNER_RUNBOOK.md` for regeneration.

## Screens

Module base `/literature`; tabs (Home auto-prepended by `BottomNav`): **Poems · Poets · Favorites ·
Settings**. Poem/Poet detail are drill-ins (not tabs).

### Poems — Home (`/literature`)

- Sticky **`SearchBar`** ("搜尋詩詞、作者…") + **`FilterToggleButton`**. Search is real-time over
  title + writer + excerpt, Traditional⇄Simplified-agnostic (`foldZh`).
- **`FilterPanel`**: 朝代 (dynasty, **single-select** pills, options from `meta.dynasties`) · 主題 / 時令 /
  選集 / 風格 (**multi-select** pills, from `meta.types` by `kind`) · **只看收藏** toggle · 清除篩選.
  Filter semantics: **OR within a kind-group, AND across groups** (e.g. (春天 OR 秋天) AND 唐詩三百首).
- **`ResultCount`** + a list of **`PoemCard`**s (title · writer · gold dynasty badge · excerpt + heart).
  Tapping a card opens the poem; the heart toggles a favourite (optimistic). Initial paint capped
  (`PAGE = 60`) with a "載入更多" button. Criteria persist via `useSessionState('wellworth:literature-home')`.

### Poem detail (`/literature/poem/:id`)

- Header: back + favourite heart. **`PoemReader`** (read-aloud) then the poem body (centred, rem-based
  display sizes), then **譯文** / **註釋** / **賞析** (`SectionCard`, only when present).
- Body loaded lazily via `getPoem(id)` (runtime-cached). Keyed by route id, so the reader resets per
  poem. The writer name links to the poet.

### Poets (`/literature/poets`) + Poet detail (`/literature/poet/:id`)

- **Poets**: writers grouped by dynasty (corpus order) as tappable chips (`groupWritersByDynasty`).
- **Poet detail**: name · dynasty · portrait (graceful fallback when `headImageUrl` is dead) · **作者簡介**
  bio · **作品** works list (links to each poem).

### Favorites (`/literature/favorites`)

- The user's favourited poems (index filtered by the favourite-id set), same `PoemCard`.

### Settings (`/literature/settings`)

- **朗讀** (read-aloud): **自動循環** toggle (`profile.literature_tts_autoloop`) + **預設語言** 粵/國
  `SegmentedTabs` (`profile.literature_tts_lang`). Auto-saves on change.

## Read-aloud (`useSpeech` + `PoemReader`)

- **`src/hooks/useSpeech.ts`** wraps the Web Speech API (`speechSynthesis`): picks a zh-HK (粵) / zh-CN
  (國) voice (voices load async via `voiceschanged`), drives `progress` from `onboundary`, seeks by
  re-speaking from a char offset, optional auto-loop, always cancels on unmount/stop.
- **`src/components/PoemReader.tsx`**: 粵/國 `SegmentedTabs`, play/stop, a 0–100% slider that doubles as
  seek. When no matching voice exists (`!voiceAvailable`) it shows a quiet note instead of failing.
- **iOS caveat (PARKED):** a zh-HK Cantonese voice is device-dependent and often absent; Mandarin is
  broadly available.

## Data model (Supabase)

### `poem_favorite` (user-owned) — `supabase/migrations/16_literature_schema.sql`

- `user_id` UUID → auth.users (ON DELETE CASCADE) · `poem_id` INTEGER (the static corpus id — **no FK**,
  the corpus isn't in the DB) · `created_at` · **PK (`user_id`, `poem_id`)**.
- RLS on; select/insert/delete-own policies (`(select auth.uid()) = user_id`); no update / no
  `moddatetime` (toggle = insert/delete). Explicit grants to `anon`/`authenticated`.

### `profile` columns — `supabase/migrations/17_literature_profile_settings.sql`

- `literature_tts_lang` TEXT default `'zh-HK'` CHECK in (`'zh-HK'`,`'zh-CN'`) · `literature_tts_autoloop`
  BOOLEAN default false.
