# 11 — Literature Module

Classical Chinese poems & prose (诗书) with search, filter, read-aloud (粵/國) and favouriting. Integrated from the standalone `tiggerkk/chinese-literature` app — **rewritten** to WellWorth conventions (dark design system, shared components, data layer); only the **read-aloud algorithm** and the **corpus data** were carried over.

## Architecture — static corpus + Supabase favourites

- The **corpus** (poems, writers, types) is **immutable, shared, non-private reference data**, so it ships as a **versioned STATIC ASSET** under `public/literature/**` (not a Postgres table): fully offline, zero DB storage/egress, updates by redeploying files. This is the one corpus where RLS and cross-device sync add nothing.
- Only the **user-owned** state is in Supabase: `poem_favorite` (a join row) + two `profile` read-aloud columns — so favourites + settings sync across devices.
- **Search/filter is client-side** over the in-memory index (the genuine departure from other modules, correct because the data is static + bundled), using the always-resident `foldZh` (Traditional⇄Simplified-agnostic — `src/lib/zh-fold.ts`).
- **Dark design system only** — the source app's light cream/brown theme was not ported.

### Static asset layout (`public/literature/`, generated by `scripts/build-literature-data.mjs`)

- `meta.json` — `{ version, types[{id,name,kind,sortOrder}], writers[{id,name,dynasty}], dynasties[] }`. `kind ∈ theme|season|anthology|style|other`; the first four are shown as filters. **Precached.** `dynasties[]` is the **distinct corpus dynasty values** (not the shared `src/constants/dynasty.ts` list), sorted oldest→newest by `DYNASTY_ORDER` in the build script — which lists **every** value the corpus emits (incl. `隋代`/`金朝`/`當代` and the catch-all `未知`, last) so the 朝代 pills render in chronological order. `DYNASTY_ORDER` only sorts; it never drops a value. The curated filter vocabulary (in `scripts/build-literature-data.mjs` `TYPE_GROUPS`) matches this corpus's tag spelling — 主題: 愛國/邊塞/懷古/羈旅/貶謫/哲理/景物/友情/愛情/孤獨/抒情/思念 · 時令: 春天/夏天/秋天/冬天/節日 · 選集: 唐詩三百首/宋詞三百首/古詩三百首/古詩十九首/詩經/楚辭/古文觀止 + 小學/初中/高中 各 古詩/文言文 · 風格: 婉約/豪放/樂府. Everything else (≈500 fine-grained gushiwen tags) stays `other`.
- **Alias layer (`TYPE_ALIAS_GROUPS`):** because most poems are tagged by fine-grained style/theme tags rather than the curated set, ~85 high-frequency `other` tags are **aliased** onto a curated parent (e.g. 寫景/山水/月亮 → 景物 · 中秋節/春節/… → 節日 · 抒懷/感慨 → 抒情 · 懷人/回憶 → 思念 · 民歌 → 樂府), so a poem tagged only with an alias still surfaces under its filter. The pills are unchanged — aliases only feed them. **The full curated lists + alias map are the source of truth in `scripts/build-literature-data.mjs` (`TYPE_GROUPS` + `TYPE_ALIAS_GROUPS`)** — edit there + rebuild to add/move a filter; this doc isn't an exhaustive copy. The unmapped long tail (misc tags like 女子/生活/故事/讚美/飲酒) stays `other`; those poems remain reachable via the unfiltered list, search, and dynasty.
- **Tag coverage is source-limited.** Only **~20%** of poems (≈2,037 / 10,000) carry **any** type tag in the source `poems.db`; the other ~80% were never tagged by theme/form, so no mapping can reach them via a type filter (they're still found by search / dynasty / unfiltered browse). After aliasing, **≈95% of the _taggable_ poems** carry a filter tag — effectively the ceiling. The build prints `coverage: N/10000` so this stays visible; chasing higher overall coverage means mis-filing, not better data.
- `index.json` — poem search/browse index: `[{ id, title, writerId, writer, dynasty, typeIds, excerpt }]` (excerpt = first 40 chars of HTML-stripped content, so a list card renders with no body fetch). **Precached** → browse/search/filter/favourites all work offline.
- `poem/<id>.json` — full body `{ …, content, translation, remark, shangxi, typeIds }`. **Runtime-cached** on open (CacheFirst, cache `literature-bodies-v1`).
- `writer/<id>.json` — `{ …, simpleIntro, detailIntro, headImageUrl, poemIds }`. **Runtime-cached.**

`poems.db` is the build input (gitignored at `scripts/literature/poems.db`); the generated `public/literature/**` is committed (the deployable source of truth — no DB / `better-sqlite3` at deploy/CI time). PWA wiring: precache via `includeAssets`; runtime-cache + `globIgnores` in `vite.config.ts`. Favouriting a poem also `cache.add()`s its body so saved poems read offline (lazy default otherwise caches only what's been opened). See `docs/OWNER_RUNBOOK.md` for regeneration.

## Screens

Module base `/literature`; tabs (Home auto-prepended by `BottomNav`): **Poems · Poets · Favorites · Settings**. Poem/Poet detail are drill-ins (not tabs).

### Poems — Home (`/literature`)

- **Search bar** ("搜尋詩詞、作者…"): matches title, writer, excerpt; Traditional⇄Simplified-agnostic (`foldZh`); **Filter button** to the right.
- **SortControl**, **只看收藏 toggle**, **Clear Filters button**: Sort over (labelled 排序) { 朝代, 作者, 標題 } with an **asc/desc** toggle; default is **朝代** descending.
- **Filter panel**: 朝代 (dynasty, **single-select** pills, options from `meta.dynasties`) · 主題 / 時令 / 選集 / 風格 (**multi-select** pills, from `meta.types` by `kind`).
  - Pills use the shared **`FilterPill`** (`text-body`, `bg-input text-text-primary` inactive / accent-filled selected).
  - Filter semantics: **OR within a kind-group, AND across groups** (e.g. (春天 OR 秋天) AND 唐詩三百首). Sorting is the pure `sortPoems` (dynasty rank = position in `meta.dynasties`, so it's corpus-consistent; author/title by `localeCompare`; null/unknown dynasty sorts last).
- Each row carries:
  - Line 1: **title · writer · gold dynasty badge**.
  - Line 2: **excerpt** + **heart** (toggle is optimistic).
  - Tap → opens the poem; **swipe-left → Delete** (optimistic).
- Initial paint capped (`PAGE = 60`) with a "載入更多" button.

### Poem detail (`/literature/poem/:id`)

- Header: **X** (close, left) · the **poem title** (`text-title`, **truncates** with `…` when long) · the linked **writer** + gold **dynasty** chip (`text-caption`/`text-section`, `shrink-0` so they survive a long title) · favourite heart (right). Body: **`PoemReader`** (read-aloud), the poem's tags as `FilterPill`s (resolved id→name from `meta.types`), then the collapsible color-accented sections via **`Collapsible`**: **原文** (always) · **譯文** / **註釋** / **賞析** (each only when present **and** visible per the Settings → 顯示 prefs). 原文 opens by default; the prose sections start collapsed. Section accent colours come from `LITERATURE_SECTION_COLOR` (constants → `--color-lit-*`).
- Body loaded lazily via `getPoem(id)` (runtime-cached); `meta` (precached) resolves tag names. Keyed by route id, so the reader resets per poem. The writer link opens the poet; **Esc** (or X) closes.

### Poets (`/literature/poets`) + Poet detail (`/literature/poet/:id`)

- **Poets**: a **curated 名家 roster** (not every writer in the corpus), grouped by dynasty (corpus order) as tappable **`FilterPill`**s (`groupWritersByDynasty`). The roster = the `FAMOUS_WRITERS` set in `scripts/build-literature-data.mjs` (50 classical poets), ported from the source app's `isFamous` flag (`migrate-writers.cjs`); only those names (with ≥1 poem) are emitted into `meta.writers`. **No screen title** (the old 名家 header was removed) — the panel sits at the standard top inset like the other tabs.
  - **F: the famous list follows THIS corpus's HK-Traditional (OpenCC) spelling, not the source's.**. The source's `高啟` is stored here as `高啓` (啓 variant), so `FAMOUS_WRITERS` uses `高啓`. The build `console.warn`s any name with no matching writer — if a future corpus rebuild drops a poet from 名家, check for an OpenCC spelling drift first.
  - Non-famous writers are still reachable: `writer/<id>.json` is written for **every** writer with poems, so tapping a non-famous author from a poem-detail page still resolves.
- **Poet detail**: header = **X** (close) · poet **name** · gold dynasty chip. Body = portrait (graceful fallback when `headImageUrl` is dead), then collapsible color-accented **作者簡介** bio (gated by the Settings → 顯示 prefs, starts collapsed) · **作品** works list (always shown, links to each poem) via `Collapsible`.

### Favorites (`/literature/favorites`)

- The user's favourited poems (index filtered by the favourite-id set), same `PoemCard`. **No screen title** (the old 收藏 header was removed) — bumped up to the standard top inset.

### Settings (`/literature/settings`)

- Header: **X** (close) · **萬卷詩書 Settings**.
- **顯示** (display, first): **可見詩書欄位** → poem-detail field visibility (譯文/註釋/賞析; `profile.literature_poem_visible_fields`, NULL = all; 原文 always shown) · **可見名家欄位** → poet-detail field visibility (作者簡介 only; `profile.literature_writer_visible_fields`; 作品 always shown). Both open the shared **`VisibleFieldsSheet`** (thin wrappers `LiteraturePoem/WriterFieldsSheet`, given a Chinese `title`).
- **朗讀** (read-aloud): **自動循環** toggle (`profile.literature_tts_autoloop`) + **預設語言** 粵/國 `SegmentedTabs` (`profile.literature_tts_lang`). Auto-saves on change.

## Read-aloud (`useSpeech` + `PoemReader`)

- **`src/hooks/useSpeech.ts`** wraps the Web Speech API (`speechSynthesis`): picks a zh-HK (粵) / zh-CN (國) voice (voices load async via `voiceschanged`), drives `progress` from `onboundary`, seeks by re-speaking from a char offset, optional auto-loop, always cancels on unmount/stop.
- **`src/components/PoemReader.tsx`**: 粵/國 `SegmentedTabs`, play/stop, a 0–100% slider that doubles as seek. When no matching voice exists (`!voiceAvailable`) it shows a quiet note instead of failing. An **info icon** (right of play) toggles an inline note explaining that iPhone exposes only the iOS default Chinese voice and how to change it (Settings → Accessibility → Read & Speak → Voices → Chinese); tap the icon again (or the note) to dismiss.
- **iOS caveat (PARKED):** iOS Safari's Web Speech API exposes only the **one** Chinese voice set as the iOS system default, so the 粵/國 toggle is inert on iPhone (both follow that default) — the info note above guides the user to switch it. Desktop browsers expose both voices, so the toggle works there.

## Data model (Supabase)

### `poem_favorite` (user-owned) — `supabase/migrations/16_literature_schema.sql`

- `user_id` UUID → auth.users (ON DELETE CASCADE) · `poem_id` INTEGER (the static corpus id — **no FK**, the corpus isn't in the DB) · `created_at` · **PK (`user_id`, `poem_id`)**.
- RLS on; select/insert/delete-own policies (`(select auth.uid()) = user_id`); no update / no `moddatetime` (toggle = insert/delete). Explicit grants to `anon`/`authenticated`.

### `profile` columns — `supabase/migrations/17_literature_profile_settings.sql`

- `literature_tts_lang` TEXT default `'zh-HK'` CHECK in (`'zh-HK'`,`'zh-CN'`) · `literature_tts_autoloop` BOOLEAN default false.
- `literature_poem_visible_fields` TEXT[] (NULL = all visible) — Poem-detail 譯文/註釋/賞析 toggles · `literature_writer_visible_fields` TEXT[] (NULL = all) — Poet-detail 作者簡介 toggle. 原文 / 作品 are always shown and never stored. Both follow the shared `VisibleFieldsColumn` / `isFieldVisible` pattern.
